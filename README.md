# Я Депутат

PWA-приложение для муниципальных депутатов — управление мероприятиями, комиссиями и участием.

## Возможности

### Админ-панель
- Управление депутатами (CRUD, коды входа)
- Управление комиссиями и их составом
- Создание мероприятий трёх типов: комиссионные, сессионные, внешние
- Загрузка файлов к мероприятиям (до 10 файлов, 20 МБ каждый)
- Отслеживание участия: ожидание → просмотрено → подтверждено / отклонено
- Push-уведомления и напоминания
- Дашборд со статистикой

### Интерфейс депутата
- Мобильный адаптивный интерфейс с нижней навигацией
- Просмотр мероприятий (предстоящие, прошедшие, все)
- Подтверждение или отклонение участия
- Скачивание прикреплённых файлов
- Push-уведомления о новых мероприятиях
- Календарь событий
- Профиль

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| Сервер | Node.js + Express |
| База данных | SQLite (better-sqlite3) |
| Аутентификация | JWT |
| Push-уведомления | Web Push (VAPID) |
| Фронтенд | Vanilla JS (SPA), HTML5, CSS3 |
| PWA | Service Worker, Web App Manifest |

## Запуск

```bash
# Установка зависимостей
npm install

# Запуск (порт 3000 по умолчанию)
node server/server.js

# Или с указанием порта
node server/server.js 8080
```

## Деплой на сервер (Ubuntu 22.04)

```bash
sudo bash deploy.sh
```

Скрипт автоматически:
- Установит Node.js 20
- Создаст системного пользователя `ya-deputat`
- Настроит systemd-сервис с автоперезапуском
- Сгенерирует JWT_SECRET

После деплоя настройте Nginx Proxy Manager для проксирования на `127.0.0.1:3000`.

## Учётные данные по умолчанию

- **Админ:** `admin` / `admin123` (смените после первого входа!)
- **Депутат:** вход по номеру телефона + 4-значный код

## API

### Аутентификация
- `POST /api/auth/login` — вход админа
- `POST /api/auth/deputy-login` — вход депутата

### Админ API (требует JWT)
- `/api/admin/deputies` — управление депутатами
- `/api/admin/commissions` — управление комиссиями
- `/api/admin/events` — управление мероприятиями
- `/api/admin/stats` — статистика

### API депутата (требует JWT)
- `/api/deputy/events` — список мероприятий
- `/api/deputy/events/:id/seen` — отметить просмотр
- `/api/deputy/events/:id/respond` — подтвердить/отклонить
- `/api/deputy/profile` — профиль
